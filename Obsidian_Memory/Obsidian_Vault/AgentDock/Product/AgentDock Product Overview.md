# AgentDock Product Overview

## Position

- Visual command centre for multi-agent work.
- See every agent. Control every handoff. Ship without losing the thread.
- Not a chatbot. Not just a task board. The operating layer between humans and AI agents.

## Core Idea

Most agent work is hidden inside terminals, chat windows, and scattered logs.

AgentDock makes the work visible:

- who is working
- what role they have
- what task they own
- what stage the task is in
- what they changed
- what needs human approval

The product turns AI agents from invisible chat sessions into an organised team.

## Product

- [[Project Dock]] is the main view — all active agents, tasks, and pipelines visible.
- [[Agent Board]] shows who is working, what role, what model, what status.
- [[Pipeline View]] is the structured workflow: PLAN → ASSIGN → RUN → TRACK → REVIEW → MERGE → SHIP.
- [[Handoff Log]] captures what each agent produced, what changed, what is blocked.
- [[Review Gate]] is the human checkpoint before merge, push, or deploy.
- [[Context Panel]] holds project memory, rules, and constraints.

## Target Users

- Solo founders running several AI tools at once.
- Developers using Codex, Claude, OpenCode, Cursor, or CLI agents.
- Agencies managing repeated client build workflows.
- Product operators who need research, build, QA, and deployment agents working in sequence.

## Problem

AI agents are powerful but messy:

- Each agent lives in a separate interface.
- Context gets lost between tools.
- Work is hard to audit.
- It is unclear which agent owns which task.
- Parallel work creates merge conflicts and duplicated effort.
- Humans cannot easily see progress without reading logs.

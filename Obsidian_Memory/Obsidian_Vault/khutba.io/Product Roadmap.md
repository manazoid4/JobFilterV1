# khutba.io Product Roadmap

> Based on competitive intelligence (8th May 2026). Focus: screen-first, unlimited, UK.

## Phase 1: Core Product (May — July 2026)

**Goal:** Ship the screen-first display that no competitor has.

### Must Ship
- [x] Screen-first display page (large text, auto-scroll, RTL)
- [x] Admin panel (mic control, language selection)
- [x] Socket.io real-time broadcast
- [x] 5 UK languages: Arabic, English, Urdu, Bengali, Somali
- [x] Landing page with screen-first positioning
- [x] Pricing page with MinbarLive comparison
- [x] WhatsApp share integration
- [x] Ramadan countdown component
- [x] UK prayer times API (Birmingham, London, Manchester, Leeds, Bradford)

### Technical
- [ ] Deepgram STT integration (server-side, better than browser STT)
- [ ] Google Translate API integration (server-side translation pipeline)
- [ ] Supabase setup (sessions, transcripts, users)
- [ ] Stripe billing (3 tiers: £29/£59/£99)
- [ ] Session management (create, join, end)

### Why This Phase
MinbarLive has 7 modules. We ship ONE thing perfectly: live translation on screen. This is the moat. If we don't ship screen-first fast, MinbarLive will add it and the moat disappears.

## What We Will NOT Build

Based on competitive analysis, these features are distractions:

1. **Podcast/Studio Mode** — Too niche. Most masjids don't need it.
2. **Meeting Translation** — Different use case, different buyer.
3. **Donation Processing** — Regulatory nightmare. Not core.
4. **Prayer Time Calculation** — Commodity feature. Many free tools exist.
5. **Mobile App** — Browser-based is fine. No one wants another app.
6. **Community Social Features** — Not the problem we're solving.
7. **Video Streaming** — Bandwidth-heavy, not core value.
8. **135+ Languages** — Quality over quantity. UK masjids need 5, not 135.

### Build Rule
A feature gets built only when:
- Requested by 5+ paying masjids
- A competitor's version is getting traction
- It directly supports the core value proposition (live translation on screen)

# UI Audit - 2026-05-09

Scope: static UI/navigation/homepage structure review only. No source edits.

## Top 8 Actionable Fixes

1. Tighten homepage to one primary conversion path.
   Target: `src/pages/HomePage.tsx` lines 57-100, 438-481, 567-590.
   Issue: Homepage repeatedly alternates between free scan, pricing, dashboard, add-ons, and waitlist. The first screen says scan/pricing, mid-page sends users to dashboard, final CTA adds waitlist form. Pick primary action: `SCAN MY AREA FREE`; keep pricing secondary.

2. Remove or demote add-on product blocks from the homepage.
   Target: `src/pages/HomePage.tsx` lines 48-53, 357-435, 486-528.
   Issue: Vantage, Vicinity, Codex, EPC, Signals, and TradieStack dilute the core lead intelligence pitch. Move them behind footer/nav or a compact "more tools" strip after conversion sections.

3. Fix duplicated "HOW IT WORKS" sections.
   Target: `src/pages/HomePage.tsx` lines 322-352 and 438-467.
   Issue: Two separate how-it-works blocks describe different flows. Combine into one sequence around `SCAN -> FILTER -> WHATSAPP -> TRACK`, with all steps linking to real lead action.

4. Fix mobile nav grid mismatch.
   Target: `src/components/TopNav.tsx` lines 83-91.
   Issue: Mobile quick actions use `grid-cols-3` but render only two links, leaving a dead third column. Use `grid-cols-2` or add the intended third action.

5. Stop the launch modal from interrupting first-page comprehension.
   Target: `src/App.tsx` line 129 and `src/components/LaunchWaitlistModal.tsx` lines 7-40.
   Issue: Global modal appears on `/` after 350ms, before users understand the product. It competes with hero CTAs and may hurt mobile. Delay until intent signal, scroll depth, or second visit.

6. Correct the lead URL shown in WhatsApp preview.
   Target: `src/pages/HomePage.tsx` line 636 and `src/App.tsx` lines 94-95.
   Issue: Preview uses `jobfilter.uk/lead/abc123`, but the app route is `/leads/:id`. This is a trust and navigation mismatch. Change example to `/leads/abc123`.

7. Replace unverifiable testimonial-style claims or label them as examples.
   Target: `src/pages/HomePage.tsx` lines 25-45 and 291-302.
   Issue: Quotes cite Reddit/Screwfix/community sources but appear hard-coded and unlinked. For trust, either link evidence, mark as examples, or replace with product-proof metrics that can be backed by data.

8. Reduce mobile vertical load and section count.
   Target: `src/pages/HomePage.tsx` lines 57-590 and `src/index.css` lines 367-430.
   Issue: Mobile CSS compresses headings/buttons, but the homepage still has many full-width sections before final form. Keep first mobile page to hero, proof lead card, simple old-vs-new, and one CTA before deeper content.

## Responsive Risks

- Very large uppercase headings are broadly clamped in `src/index.css` lines 369-372, but local inline styles in `HomePage.tsx` still create heavy blocks on small screens.
- `.jf-button` becomes full-width under 640px in `src/index.css` lines 422-431; stacked CTAs are clear, but repeated CTA pairs add scroll fatigue.
- Sticky nav plus modal overlay can consume the mobile first viewport before the value proposition lands.

## Navigation Notes

- Top nav focuses on `Find Jobs`, `Pipeline`, `Pricing`, `Signals`, `Free Tools`; homepage also heavily promotes `Vantage`, `Vicinity`, `Codex`, `EPC`, and `TradieStack`. Align nav and homepage around the core revenue path.
- Footer has a clean product/tools split, so add-ons can live there instead of competing in the homepage body.

# Claude Design Research - 2026-05-09

## Findings

- Claude Design and related GitHub design-system workflows work best when they extract strict tokens, spacing, typography, and components from real product material.
- Generic prompts create generic SaaS dashboards. JobFilter needs a documented tactical system: black, DeWalt yellow, steel greys, hard borders, compact panels, and obvious action states.
- Good operational dashboards prioritise current state, source confidence, urgency, and next action over decorative layout.
- Mobile field interfaces need fewer choices, larger tap targets, and short CTA labels.

## Adaptation For JobFilter

- Use territory ownership as the emotional hook: secure the patch before another firm does.
- Use intelligence language only where it means something concrete: source, score, urgency, estimated value, trade match.
- Avoid generic UI sections competing for attention on the homepage.
- Treat homepage as a command screen: explain value, show signal proof, route to scan or claim.

## Design Rules Added

- Sharp edges and 2px black borders.
- Yellow reserved for CTA, status, and tactical highlights.
- Dense signal cards instead of lifestyle imagery.
- Mobile nav first actions: Scan and Claim.
